GIT:
- Para iniciar git --> clicar com direito na pasta de destino e "Git Bash Here". Caso atalho não funcionar, abrir manualmente o software e chamar os caminhos, com o comando "cd", ex: 'cd d:'

Branch -> Ramificação (versões) - utilizado para não corromper ou configurar de forme errada o código fonte principal.

GIT serve para versionamento local. GITHUB serve para versionamento on-line e para divulgação de códigos abertos, etc ("rede social do TI).

No git inicialmente sempre te joga para o branch Master, no Github sempre no Main, isso pode ser um problema para alterações simultaneas. A solução mais fácil é mudar o branch no Git, com o comando -> 'git branch -M main' (te joga para o branch main).
 -git add . -> para adicionar todos os arquivos da pasta de destino no git.
